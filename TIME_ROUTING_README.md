# רכיב ניתוב לפי שעות (Time Routing)

## תיאור
רכיב חדש שמאפשר לנווט את התזרים של הבוט בהתאם לשעה הנוכחית.

## שימוש

### הוספת הרכיב
1. גרור את הרכיב "ניתוב לפי שעות" מתוך קטגוריית "פעולות מערכת" בסיידבר
2. הרכיב יופיע עם אופציית ברירת מחדל אחת

### הגדרת תנאי שעות
כל תנאי שעה כולל:
- **משעה**: שעה התחלתית (0-23)
- **עד**: שעה סיומית (0-23)
- **טקסט להצגה**: (אופציונלי) תיאור התנאי

### אופציית ברירת מחדל
- אופציית ברירת המחדל תמיד קיימת
- היא תופעל כאשר השעה הנוכחית לא תואמת לאף אחד מתנאי השעות האחרים

### דוגמה לשימוש

#### תרחיש: שעות פעילות משרד
משרד פתוח בין 8:00 ל-16:00:

1. **תנאי ראשון** (שעות פעילות):
   - משעה: 8
   - עד: 16
   - טקסט: "שעות פעילות"
   - חיבור: לרכיב "הודעת טקסט" עם הטקסט "פניתך הועברה לנציג"

2. **ברירת מחדל** (מחוץ לשעות):
   - חיבור: לרכיב "הודעת טקסט" עם הטקסט "משרדנו סגור כעת. אנא נסה שוב בין 8:00 ל-16:00"

### הוספת תנאים נוספים
לחץ על "הוסף טווח שעות" להוספת תנאי נוסף.

### מחיקת תנאים
לחץ על סימן ה-X ליד התנאי למחיקתו (לא ניתן למחוק את ברירת המחדל).

## טכני

### מבנה הנתונים
```json
{
  "type": "action_time_routing",
  "data": {
    "options": ["default", "פניתך הועברה לנציג"],
    "timeRanges": [
      { "fromHour": 8, "toHour": 16 }
    ]
  }
}
```

### לוגיקת הפעולה
1. הרכיב מקבל את השעה הנוכחית מהשרת
2. בודק אם השעה נמצאת בתוך אחד מטווחי השעות המוגדרים
3. אם נמצא התאמה - ממשיך לתזרים המחובר לאופציה זו
4. אם לא נמצא התאמה - ממשיך לתזרים המחובר לברירת המחדל

### התנהגות השעות
- השעות מוגדרות בפורמט 24 שעות (0-23)
- השעה מתבססת על שעת השרת
- הבדיקה כוללת את שעת ההתחלה והסיום (>=, <=)
